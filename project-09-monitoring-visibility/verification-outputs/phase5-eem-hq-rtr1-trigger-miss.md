# Project 09 - Phase 5 EEM HQ-RTR1 Trigger Miss

Date: 2026-05-17

## Result

The `HQ-RTR1` EEM applet was registered successfully, but the first trigger test did not fire.

## Evidence

Registered applet:

```text
event manager applet P09_EEM_LOOP99_DOWN
 event syslog pattern "Interface Loopback99, changed state to administratively down"
 action 1.0 syslog priority warnings msg "P09_PHASE5_EEM_LINK_DOWN_DETECTED on HQ-RTR1 Loopback99"
```

Trigger test:

```text
interface Loopback99
 shutdown
 no shutdown
```

Verification:

```text
show event manager history events
```

returned no events.

Local log evidence showed this actual IOS message form:

```text
%LINEPROTO-5-UPDOWN: Line protocol on Interface Loopback99, changed state to up
```

## Interpretation

The EEM policy did not fire because the syslog pattern did not match the message format generated by this IOS image.

## Fix

Update the applet pattern to match the actual `%LINEPROTO-5-UPDOWN` wording:

```text
event syslog pattern "Line protocol on Interface Loopback99, changed state to down"
```

Then trigger `shutdown`, wait a few seconds, and `no shutdown`.
